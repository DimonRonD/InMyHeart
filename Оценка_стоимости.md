# Оценка стоимости — AI-ассистент INMYHEART

**Проект:** RAG-ассистент медицинской клиники INMYHEART  
**Период расчёта:** 12 месяцев  
**Дата расчёта:** 19.05.2026  
**Курс для ориентира:** 1 USD ≈ 92 ₽ (уточняйте на дату оплаты)

Документ подготовлен для выпускного проекта (урок PEf04) и опирается на фактическую архитектуру репозитория: классификатор → RAG → quality check → эскалация (`rag/assistant.py`, `prompts.md`).

---

## 1. Исходные допущения по нагрузке

| Параметр | Значение | Комментарий |
|----------|----------|-------------|
| Число офисов / точек приёма | **20** | Единая сеть клиники, один backend |
| Смена администратора | **12 ч** | Один сменный администратор на офис |
| Интенсивность обращений | **1 запрос / 5 мин** | Телефон, Telegram, стойка регистратуры |
| Запросов на офис за смену | **~150** | 12 ч × 60 мин / 5 мин = 144 → округление до 150 |
| Запросов в день (сеть) | **3 000** | 20 офисов × 150 |
| Рабочих дней в году | **300** | ~6 дней в неделю с учётом праздников |
| **Запросов в год (базовый сценарий)** | **900 000** | 3 000 × 300 |
| Пиковый сценарий (365 дней) | **1 095 000** | Для чувствительности — см. §6 |

> **Важно:** 150 обращений/смену — это **весь** входящий поток на регистратуру. Не все сообщения идут в LLM: часть обрабатывается оператором напрямую (медицинские вопросы, ПДн, сложные кейсы). В §4 заложена реалистичная доля автоматизации **70%** — остальное остаётся на администраторе.

## 3. Расход токенов на один диалог

Пайплайн MVP делает **несколько вызовов LLM** на одно обращение:

| Шаг | Когда | Модель (рекомендация) | Input, ток. | Output, ток. |
|-----|-------|----------------------|-------------|--------------|
| 1. Классификатор | Всегда | nano | ~450 | ~5 |
| 2. RAG-ответ | Категория ORG, есть контекст | mini | ~2 900 | ~150 |
| 3. Quality check | ORG + включён `ENABLE_QUALITY_CHECK` | nano | ~2 650 | ~10 |
| 4. Ответ PII / MED / OTHER | Без RAG или с эскалацией | nano | ~400–800 | ~120–200 |
| 5. Резюме для оператора | При эскалации (~25% диалогов) | nano | ~700 | ~180 |

**Средневзвешенный расход на 1 обращение** (по маршрутам из `test_questions.md`: ~65% ORG+RAG, ~8% PII, ~17% MED/RESULTS/OTHER, ~10% эскалация ORG):

| Показатель | Значение |
|------------|----------|
| Средний input | **~4 700** токенов |
| Средний output | **~195** токенов |
| LLM-вызовов | **~2,8** на диалог |

**Embeddings (поиск в Chroma):**

| Операция | Токенов | Частота |
|----------|---------|---------|
| Embedding запроса пациента | ~80 | ~85% диалогов (ORG + поиск) |
| Переиндексация базы (~242 чанка) | ~100 000 | 4 раза в год |

## 10. Экономический эффект (не входит в прямой счёт AI)

Оценка для обоснования проекта на защите:

| Показатель | Расчёт |
|------------|--------|
| Обращений в год (весь поток) | 900 000 |
| Доля, закрываемая AI (70%) | 630 000 |
| Среднее время администратора на справку | ~3 мин |
| Экономия времени | **31 500 человеко-часов / год** |
| Эквивалент FTE (1 800 ч/год) | **~17,5 штатных единиц** администратора |

Прямые расходы на AI (~265 000 ₽/год) несопоставимо малы относительно фонда оплаты 20 сменных администраторов — ассистент **не заменяет** людей полностью, но снимает ~70% рутинных справочных запросов и снижает очередь на линии.

## 11. Риски и запас по бюджету

| Риск | Влияние на cost | Митигация |
|------|-----------------|-----------|
| Рост тарифов OpenAI | +10–20% | Резерв 12–15%; мониторинг usage dashboard |
| Увеличение числа LLM-вызовов (новые проверки) | +15–30% | Отключение LLM quality check там, где достаточно программной |
| Пиковые дни (сезон гриппа, акции) | +20% в отдельные месяцы | Rate limit, очередь, приоритет ORG |
| Переход на 100% автоматизации | +43% к API | Поэтапное включение каналов |
| Курс USD/RUB | ±10% в ₽ | Оплата с запасом; резервный бюджет |

## 12. Что не включено в расчёт

- ФОТ разработчиков и сопровождения (внутренняя команда / аутсорс).
- Лицензии МИС, CRM, телефонии.
- SMS-уведомления, если добавятся позже.
- Юридическое согласование обработки данных (152-ФЗ).

## 13. Рекомендации для MVP → продакшен

1. **Модели:** классификатор и quality check — `gpt-5.4-nano`; RAG-ответ — `gpt-5.4-mini` (сценарий C).
2. **Embeddings:** `text-embedding-3-small` — достаточно для ~242 чанков.
3. **Инфра:** один VPS 8 GB; Chroma и SQLite на том же сервере.
4. **Контроль расходов:** лимиты в OpenAI Dashboard, лог `kpi_metrics` в SQLite, алерт при >$300/мес.
5. **Бюджет на год с запасом:** закладывать **~300 000 ₽** (~$3 300) — покрывает гибрид, инфру и 15% резерв.
