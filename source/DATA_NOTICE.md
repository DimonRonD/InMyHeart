# Данные базы знаний INMYHEART — важно для отчёта и диплома

## Статус данных

| Аспект | Текущее состояние |
|--------|-------------------|
| **Тип контента** | **Синтетический (демо)** — вымышленные, но согласованные тексты |
| **Назначение** | Прототип RAG, тестирование пайплайна, дипломный MVP |
| **Соответствие реальной клинике** | Структура и форматы соответствуют ТЗ; **факты (цены, телефоны, расписание) не являются официальными** |
| **Продакшен** | Перед запуску у пациентов замените файлы в `source/` на **актуальные регламенты клиники** и выполните переиндексацию |

## Что нужно заменить перед продакшеном

1. FAQ, расписание, прайс-лист — из регистратуры / администрации  
2. Памятки и подготовка к анализам — утверждённые PDF/DOCX клиники  
3. Контакты (+7 …), адрес, режим работы — только из официальных источников  
4. После замены: `python scripts/index_knowledge_base.py --reset`

## Для отчёта / пояснительной записки

> База знаний `source/` на этапе MVP содержит **сгенерированные документы** (скрипт `generate_source_docs.py`), имитирующие полный набор форматов по ТЗ. Это позволяет воспроизводимо тестировать RAG, маршрутизацию и эскалацию без публикации реальных регламентов клиники. Переход на боевые данные — отдельный этап внедрения.

Подробнее о структуре папок — [README.md](README.md) в этой директории.
